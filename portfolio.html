  2. Barebones Pokémon FireRed Clone

    Date Created

    April 20, 2016

    Last Updated

    May 4, 2016

    Description

    In this project, I, with the help of Ben Richards, designed and implemented a Pokémon FireRed clone on an embedded processor with a small LED monitor. I designed and built the external game controller as well as the PokéBattle interface.

    We ran into a couple of challenges when implementing our clone. The biggest being the memory restrictions on the TM4C. The TM4C only had 256 KB of disk memory. An average sprite sheet that we used to draw our characters took up somewhere around 30-50 KB of memory. This meant we could only use 5-8 sprite sheets to represent all of the characters, pokémon, items, environments, etc. in our game.

    To comply with this constraint, we used vector graphics when possible. This allowed us to focus our sprites on the pokémon in the game...
